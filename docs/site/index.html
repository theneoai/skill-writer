Skill Definition

SKILL.md Format

YAML-based skill specification with sections, triggers, and evaluation criteria.

example-skill/SKILL.md
name: example-skill version: "1.0.0" description: en: "A sample skill demonstrating the format" zh: "示例技能，展示格式" # Core sections that define skill behavior sections: introduction: description: "What this skill does" content: "..." # Triggers that activate the skill triggers: patterns: - "create a skill" - "make a new skill" # Evaluation criteria for certification evaluation: criteria: - name: "clarity" weight: 0.3 - name: "completeness" weight: 0.4
Operation Modes

Six Modes for Skill Lifecycle

Each mode serves a specific purpose in the skill development process.

CREATE Skill Creation
▼
Generate new skills from natural language descriptions. Supports inheritance from existing skills and automatic section generation.
EVALUATE Comprehensive Evaluation
▼
Multi-dimensional scoring with GEPA trajectory analysis, SAE survivability testing, and ground truth benchmarks. Produces detailed certification reports.
LEAN Fast Heuristic Evaluation
▼
Lightning-fast evaluation (~0s, ~$0) using heuristic-based scoring for rapid iteration during development.
RESTORE Skill Repair
▼
Automatic diagnosis and repair of skill issues. Uses trajectory analysis to identify failure modes and apply corrections.
SECURITY OWASP AST10 Audit
▼
Comprehensive security analysis against OWASP AST10 guidelines. Detects command injection, path traversal, secret exposure, and other vulnerabilities.
OPTIMIZE Self-Evolution
▼
BOAD/ROAD-powered autonomous improvement. Uses UCB1 exploration and reward shaping to discover optimal skill configurations.
